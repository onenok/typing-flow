# Typing Flow 測試與驗證清單

本文檔提供完整的測試流程，確保所有功能正常工作。

## 測試前準備

### 1. 環境配置
- [ ] 安裝所有依賴：`npm install`
- [ ] 配置 `.env.local` 文件
- [ ] 執行數據庫遷移 SQL
- [ ] 確認 Supabase 項目正常运行

### 2. 啟動應用
```bash
npm run dev
```
訪問 http://localhost:3000

## 功能測試清單

### 認證系統

#### 註冊功能
- [ ] 訪問首頁，點擊「立即註冊」鏈接
- [ ] 填寫註冊表單（全名、郵箱、密碼）
- [ ] 提交後應自動登入並跳轉到首頁
- [ ] 導航欄應顯示用戶郵箱和「登出」按鈕
- [ ] 檢查 Supabase Auth 控制台，確認用戶已創建

#### 登入功能
- [ ] 點擊「登出」按鈕
- [ ] 訪問 `/auth/login` 或點擊導航欄「登入」
- [ ] 使用已註冊的郵箱和密碼登入
- [ ] 登入成功後應跳轉到首頁
- [ ] 導航欄應顯示用戶郵箱

#### 登出功能
- [ ] 點擊導航欄的「登出」按鈕
- [ ] 應跳轉到首頁，導航欄顯示「登入/註冊」鏈接
- [ ] 嘗試訪問受保護頁面（如 `/practice`），應重定向到登入頁

### 練習模式

- [ ] 登入後點擊導航欄「練習」
- [ ] 頁面應顯示練習文本
- [ ] 開始輸入，確認：
  - [ ] 輸入的字符實時顯示
  - [ ] 正確字符變為綠色
  - [ ] 錯誤字符變為紅色
  - [ ] 速度和準確率實時更新
- [ ] 完成全部文本輸入
- [ ] 應自動保存練習結果
- [ ] 顯示「查看結果」按鈕
- [ ] 點擊按鈕，應跳轉到結果詳情頁
- [ ] 結果頁顯示正確的統計數據

### 測驗模式

- [ ] 點擊導航欄「測驗」
- [ ] 頁面顯示計時器（60秒）
- [ ] 點擊「開始測驗」按鈕
- [ ] 確認：
  - [ ] 計時器開始倒計時
  - [ ] 可以輸入文本
  - [ ] 速度和字符數實時更新
- [ ] 等待計時器歸零或完成文本
- [ ] 確認結果已保存
- [ ] 可以查看結果詳情

### 結果頁面

#### 結果列表（`/results`）
- [ ] 登入後訪問 `/results`
- [ ] 如果沒有記錄，顯示「還沒有練習記錄」消息
- [ ] 有記錄時：
  - [ ] 顯示統計卡片（總會話數、平均速度、平均準確率、最佳速度）
  - [ ] 顯示所有練習記錄表格
  - [ ] 表格包含：日期、模式、速度、準確率、持續時間、操作按鈕
  - [ ] 點擊「查看詳情」按鈕，跳轉到詳情頁

#### 結果詳情（`/results/[id]`）
- [ ] 從結果列表點擊任意記錄的「查看詳情」
- [ ] 頁面顯示：
  - [ ] 總結卡片（模式、速度、準確率、持續時間）
  - [ ] 字符統計（總字符數、正確字符、錯誤次數）
  - [ ] 練習文本（正確字符綠色，錯誤紅色）
  - [ ] 詳細記錄表格（位置、期望字符、輸入字符、狀態）
- [ ] 點擊「返回結果列表」按鈕，返回列表頁
- [ ] 點擊「再次練習」按鈕，跳轉到練習頁

### 個人資料頁面

- [ ] 點擊導航欄用戶郵箱，訪問 `/profile`
- [ ] 頁面顯示：
  - [ ] 頭像（默認或自定義）
  - [ ] 用戶名稱和郵箱
  - [ ] 全名輸入框（預填充）
  - [ ] 頭像網址輸入框（預填充）
- [ ] 修改資料並提交
- [ ] 確認更新成功，無錯誤消息
- [ ] 刷新頁面，確認資料已保存

### 導航測試

- [ ] 測試所有導航鏈接是否正常工作：
  - [ ] 首頁 → `/`
  - [ ] 練習 → `/practice`
  - [ ] 測驗 → `/quiz`
  - [ ] 結果 → `/results`
  - [ ] 個人資料 → `/profile`（需登入）
  - [ ] 登入 → `/auth/login`（需登出）
  - [ ] 註冊 → `/auth/register`（需登出）
- [ ] 確認導航時頁面無刷新（SPA 體驗）
- [ ] 確認登入後導航欄狀態正確更新

### 響應式設計

- [ ] 調整瀏覽器窗口大小
- [ ] 確認佈局在不同屏幕尺寸下正常顯示
- [ ] 移動端視圖（< 768px）：
  - [ ] 導航欄是否適配
  - [ ] 卡片是否單列顯示
  - [ ] 表格是否可橫向滾動

### 錯誤處理

- [ ] 未登入時訪問受保護頁面，應重定向到登入頁
- [ ] 提交表單時網絡中斷，應顯示錯誤消息
- [ ] 數據庫操作失敗，應在控制台看到錯誤日誌
- [ ] 刷新頁面時，認證狀態應保持

## 數據庫驗證

### 在 Supabase 控制台檢查

1. **Table: users**
   - [ ] 新註冊用戶應自動創建記錄
   - [ ] 更新個人資料時，記錄應更新

2. **Table: typing_sessions**
   - [ ] 完成練習/測驗後，應創建新記錄
   - [ ] 字段值正確：user_id、mode、text_type、text_content、duration_seconds、total_chars、correct_chars、errors、wpm、accuracy

3. **Table: typing_details**
   - [ ] 每個會話應創建對應的詳細記錄
   - [ ] 字符級數據正確

4. **函數: get_user_stats**
   - [ ] 執行 `SELECT get_user_stats('user-uuid-here')` 應返回 JSON 統計數據

## 性能檢查

- [ ] 頁面加載速度正常（< 2秒）
- [ ] 打字輸入無延遲
- [ ] 頁面切換流暢
- [ ] 無控制台錯誤或警告

## 已知限制

1. **打字詳細記錄時間戳**：當前 `time_ms` 字段設置為 0，實際應用中應記錄每個字符的擊鍵時間
2. **練習文本**：當前使用固定樣本，可以擴展為從數據庫隨機選擇
3. **測驗模式**：60秒後自動結束，但用戶可能在時間結束前完成文本
